9.3. Rebooting the System

至此，所有的软件就业已安装完毕，是时候重启你的电脑了。然而，你也应该注意 一些事情。通过学习本书建立起来的系统属于最小系统，这也就意味着可能会缺失 一些你需要的功能。这也就意味着你还需要做些事情。By installing a few extra packages from the BLFS book while still in our current chroot environment, you can leave yourself in a much better position to continue on once you reboot into your new LFS installation. 以下给出了一些建议：

  • 文本模式的浏览器，例如 Lynx，可以在虚拟终端中访问 BLFS 书，以进行后续的编译打包工作。

  • GPM（GPM：A mouse server for the console and xterm） 软件包可以让 你在虚拟终端中更好的执行复制/粘贴工作。

  • 如果静态 IP 配置不能很好的适用于你当前环境的网络配置，可以安装 dhcpcd 或者是 dhcp 的客户端部分来解决。

  • 安装 sudo，可以在非 root 用户环境下编译软件包，且可以很轻松的在新系统中 安装软件。

  • 如果你想要通过舒适的 GUI 环境远程访问新系统，请安装 openssh 和它的依赖 openssl。

  • 为了更加便利的从网络中下载文件，请安装 wget。

  • 如果你有 GUID 分区表（GPT） 类型的磁盘，你也许需要 gptfdisk 或是 parted。

  • 最后，检查以下的配置文件是不是都是正确的吧。

    • /etc/bashrc

    • /etc/dircolors

    • /etc/fstab

    • /etc/hosts

    • /etc/inputrc

    • /etc/profile

    • /etc/resolv.conf

    • /etc/vimrc

    • /root/.bash_profile

    • /root/.bashrc

哎呀，劳累了那么久，赶快重启到我们那闪闪发亮的全新的 LFS 系统吧！ 首先，请退出 chroot 环境： 

logout

然后卸载虚拟文件系统： 

umount -v $LFS/dev/pts
umount -v $LFS/dev
umount -v $LFS/run
umount -v $LFS/proc
umount -v $LFS/sys

卸载 LFS 文件系统本身： 

umount -v $LFS

如果还建立了其它的挂载点，请在卸载 LFS 文件系统之前先卸载它们： 

umount -v $LFS/usr
umount -v $LFS/home
umount -v $LFS

至此，重启系统吧： 

shutdown -r now

这里假设 GRUB 引导器已经如前文所述安装完毕且配置正确，启动项 也已经自动设置为 LFS 7.7-systemd。

重启后，LFS 便已经可以使用了，你可以安装一些其它的软件以满足自己的需求。
